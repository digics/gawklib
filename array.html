        ARRAY JOINING

 

 

                 B[ 0 ] = "b0"

A[ 1 ] = "a1"        ; B[ 1 ] = "b1"

A[ 2 ] = "a2"        ; B[ 2 ] = "b2"

A[ 3 ] = "a3"        ; B[ 3 ] = "b3"

 

 

_reta( A )                == "a1a2a3"                # A[1] A[2] A[3]

 

_retad( A, -1 )        == "a3a2a1"                # A[3] A[2] A[1]

 

_retas( A, "-" )        == "a1-a2-a3"        # A[1] "-" A[2] "-" A[3]

 

_retab( A, B )        == "b0a1b1a2b2a3b3" # B[0] A[1]B[1] A[2]B[2] A[3]B[3]

 

_retabs( A, B, "-" ) == "a1b1-a2b2-a3b3" # A[1]B[1]"-"A[2]B[2]"-"A[3]B[3]

 

 

        - joining elements of an array:

 

 

                split( "abc", A, "" )

 

                _reta( A ) == "abc"

 

 

        - joining elements of an array in reversed order:

 

                split( "abc", A, "" )

 

                _retad( A, -1 ) == "cba"

 

 

        - joining elements of an array using separator:

 

                split( "abc", A, "" )

 

                _retas( A, "-" ) == "a-b-c"

 

 

        - joining elements of the two arrays:

 

                patsplit( t, A, /regexp/, B )

 

                _retab( A, B ) == t

 

        - typical patsplit, processing loop and join back:

 

                t == source string

 

                if ( patsplit( t, T, /regexp/, D ) ) {

 

                        for ( t = 1; t in T; t++ )

 

                                T[ t ] = ... processing T[ t ]

 

                        t = _retab( T, D ) }

 

                t == processed source string

                "" in D

 

                while ...

 

                        ...D[ length( D ) ] = ... peace of result

FUNCSET

 

 

        library contains five very important functions for array

        joining making awk programming much easier and effective. the

        programming design is using an elements of linear programming

        solutions with the primal priority to fastly operates with a

        large amount of indexes and the big data sizes.

 

        please note that all parameters for a functions implemented

        below have the same common parameter determination system

        allowing to omit or providing its in a relative form.

 

        see PARAMETERs DETERMINATION for details.

 

        the programming environment include special definition called

        as hids:

 

        BEGIN_

        END_

        STEP_

        NAMA_

        NAMASTRICT_

 

        hid is the global variable with an unique strong value

        assigned. the typical usage of hids is as special indexes in an

        arrays or as the special values meaning something more than

        just a value.

 

        see HIDs for details

 

        _reta( A, a, q, x, Sep, b, y  )       

 

        _retas( A, Sep, a, q, x, b, y )       

 

        _retad( A, x, a, q, Sep, b, y )       

 

                A        - source array A

                a        - start index a

                q        - number of elements to join

                x        - index step a

                Sep        - separator or

                         source array B:

 

                                b        - start index b

                                y        - index step b

                joins `q of an elements of an array `A with numeric

                indexes starting from `a with using index step `x and

                with the separator `Sep:

 

                note: all three functions do the same job and differs

                        from each other only by parameter's order.

 

                note: if `Sep is an array then function call will be        

                        redirected to _retab( A, Sep, a, q, b, x, y )

 

                example 1:

 

                        join 3 elements starting from index 2 without an        

                        separator and using index step +1:

 

                        _reta( A, 2, 3, 1 )

 

                        _retad( A, 1, 2, 3 )

 

                        _retas( A, "", 2, 3, 1 )

 

                        will cause to return like:

 

                                return A[2] A[3] A[4]

 

                example 2:

 

                        join 3 elements starting from index 2 without an        

                        separator and using index step -1 (reversed order):

 

 

                        _retad( A, -1, 2, 3 )

 

                        _reta( A, 2, 3, -1 )

 

                        _retas( A, "", 2, 3, -1 )

 

                        will cause to return like:

 

                                return A[2] A[1] A[0]

 

                example 3:

 

                        join 3 elements starting from index 2 with separator "-"

                        and using index step -1:

 

                        _retas( A, "-", 2, 3, -1 )

                        _reta( A, 2, 3, -1, "-" )

 

                        _retad( A, -1, 2, 3, "-" )

 

                        will cause to return like:

 

                                return A[2] "-" A[1] "-" A[0]

 

 

        _retab( A, B, a, q, b, x, y, Sep )

 

        _retabs( A, B, Sep, a, q, b, x, y )

 

                A        - source array A

                B        - source array B

                a        - start index a

                q        - number of elements to join

                b        - start index b

                x        - index step a

                y        - index step b

                Sep        - separator

 

 

                joins `q of element-pairs of an arrays `A and `B with

                numeric indexes starting from `a/`b and using index step

                in `x/`y for array `A/`B appropriately and with the        

                separator `Sep:

 

                note: both functions do the same job and differs from

                        each other only by parameter's order

 

                note: _retab(): if `Sep isn't equals to empty string ( !=

                "" ) then functon call will be redirected to

                _retabs( A, B, Sep, a, q, b, x, y )

 

                note: _retabs(): if `Sep equals to empty string ( != "" )

                then functon call will be redirected to

                _retab( A, B, a, q, b, x, y )

 

                example 1:

 

                        _retab( A, B, 1, 3, 5, 1, -1 )

 

                        will cause to return like:

 

                                return A[1] B[5] A[2] B[4] A[3] B[3]

 

                example 2:

 

                        _retabs( A, B, "-", 1, 3, 5, 1, -1 )

 

                        will cause to return like:

 

                                return A[1] B[5] "-" A[2] B[4] "-" A[3] B[3]

 

                _retab() was designed as the opposite of the built-ins

                split() and patsplit() functions that are called with the

                four parameters.

 

                here is the typical usage example:

 

                example:

 

                        # typical patsplit, processing loop and joinback

 

                        if ( patsplit( t, T, /.../, D ) ) {

 

                                for ( t = 1; t in T; t++ )

 

                                        T[ t ] = ... T[ t ] ...        # processing

 

                                t = _retab( T, D ) }

 

                        ...

 

 

        _defa( i, A )       

 

                defines `A[ `i ] as an clear array.

 

                if `A[ `i ] is already defined as an array then _defa()

                always clear it.

 

                returns `i.

 

                example:

 

                # delete T

                patsplit( ..., T[ _defa( "A", T ) ], ..., T[ _defa( "B", T ) ]

 

 

        _leta( i, A )       

 

                defines `A[ `i ] as an array.

 

                if `A[ `i ] is already defined as an array then nothing

                is doing.

 

                returns `i.

 

                example:

 

                        -

 

 

        _dela( A, v )       

 

                clear array `A from inside.

 

                there is difference between using statement:

 

                        delete `A[ `i ]

 

                and a function call:

 

                        _dela( `A[ `i ] )

 

                the last one do not deleting index `i from array `A while

                delete statement will remove index `i from array `A.

                this may be useful in case if for example array `A is the

                SYMTAB array that is stricted for elements deleting.

 

                returns `v.

 

                example:

 

                        -

 

 

        _delete( i, A )       

 

 

                delete index `i from array `A.

 

                allows user to perform deleteing index from an array        

                without using statement.

 

                also in gawk version before 5.x this function allows to

                bypass  deleting index protection from arrays                

                SYMTAB/FUNCTAB. for gawk 5.x it is no longer actual cause

                now SYMTAB have the dynamic protection (but it's not for

                all kind of operations ;) )

                returns `i.

PARAMETERs DETERMINATION

 

 

        all array-joining functions use a common parameter

        determination system located         in the _reta_startlen() function.

 

        the operation of this function is described         below.

 

        if user wants to creates an it's own special array functions

        then are may use         _reta_startlen() function to pre-determinate

        a parameters.

 

        the following is the detailed explanation of the how

        _reta_startlen() works.

 

        see also ARRAY TYPES

 

        the parameters determination process starts by getting

        information about a given array(s) and storing its in the

        special set of the global vars:

 

                _reta_tp                - type: array type ( C/B )

 

                                                C        - COMMON type array

 

                                                B        - BEGIN/END type array

 

                                                see ARRAY TYPES

                _reta_first                - first: index of the first index in a

                                        given array

 

                _reta_last                - last: index of the last index in a

                                        given array

 

                _reta_ax                - arrstep: array index step

 

        the arrstep - array index step - may be determinated by

        `AB[ STEP_ ] if it's                 exist in a given array:

 

 

                - if STEP_ is exist in a given array then it's contains

                exact array index step:

 

                        arrstep = `AB[ STEP_ ]

 

                - otherwise arrstep is equals to 1*

 

                        * there is some exception described in ARRAY TYPES

 

 

                do not confuse the arrstep and the index step parameter

                provided when calling an array-join functions.

 

                although if when calling such a function the index step

                parameter is omitted ( == "" ) then its will be equals to

                the array index step.

 

                the arrstep is actually index step that is using while

                write-to-array operations performed.

 

                the idxstep - index step parameter that's will be

                consider further - provided to an array joining functions

                is using it's while read-from-array operations performed.

 

                see below for a details

 

 

                the first index of an array is may contained at

                `AB[ BEGIN_ ] if it's exist in an array:

 

                - if BEGIN_ is exist in a given array then its contains

                first array's index:

 

                        first = `AB[ BEGIN_ ]

 

                - otherwise first index of an array will be calculated as

                the follows:

 

                        first = ! ( 0 in `AB )

 

                        that is: the first index in a given array  will be 0 if

                        the index 0 is existed in a given array. otherwise it's

                        will be equal to 1

 

 

        the last index of an array may be determinated by `AB[ END_ ]

        if it's existed in an array that is the pointer beyond the last

        element of an array ( == last + arrstep ):

 

                - if END_ is exist in a given array then last index of a

                given array will be calculated as follows:

 

                        last = `AB[ END_ ] - arrstep

 

                - otherwise last index of an array will be calculated as

                the follows:

 

                        last = first + ( _reta_length( `AB ) - 1 ) * arrstep

 

                        note: that _reta_length returns number of indexes

                        in a given array regardless of the presence or

                        absence of the following indexes inside it:

 

                                BEGIN_

                                END_

                                STEP_

 

 

`x/`y         - idxstep: index step parameter for array `A/`B

 

       

        next,the parameter determination system needs to know the

        `idxstep - index step parameter while using it as the current

        index pointer modifier while joining an elements

 

        note that index step value is determinated by the _reta_start()

        function that is         placing the results in the globvar _reta_x

 

 

        array v3 supports index step parameter that is adding to `a/`b

        after each read-from-array operation is performed:

 

                ... A[a]B[b] A[a+=x]B[b+=y] ...

        `xy may be any number including negatives that allows to join

        array's elements in forward or reversed order

 

        if `xy is omitted ( == "" )  then it's will be equals to

        default index step. the default index step is always equals to

        1 except in case if STEP_ is exist in a         given array. if thus

        then default index step will be loaded from `AB[ STEP_ ].

       

        note: that there is another exception when default index step

        may be equals to -1. see ARRAY TYPES for details.

 

        if `xy is present ( != "" ) then its will be converted to a

        number:

 

                _reta_x = 0 + `xy

 

 

 

`a/`b         - start index parameter for array `A/`B

 

 

        then parameters determination system is calculating the start

        index of the join process.

        `ab determinates start index of the join process.

 

        note that start index value is determinated by the

        _reta_start() function that is placing the results in the

        globvar _reta_a

 

        `ab may be omitted ( == "" ) and then it's will be targeted at

        the first or last (depending from a given index step polarity)

        index of a given array:

 

                - if index step polarity is positive ( >= 0 ) then `ab

                will be equals to the first array index (_reta_first)

 

                - if index step polarity is negative ( < 0 )  then `ab

                will be equals to the last array index (_reta_last)

 

        `ab can also contain the leading character "^" or "$" followed

        by an optional decimal number. in this case, it's the relative

        form of the start index. it is calculated by skipping the

        specified number of indexes counted from the first or

        last (depending on index step polarity) index of a given array:

 

                example 1:

                A[ 0 ] = a                # $3 or ^0

                A[ 1 ] = b                # $2 or ^1

                A[ 2 ] = c                # $1 or ^2

                A[ 3 ] = d                # $0 or ^3

 

                _reta( A, "$2" )        == "bcd"

 

                        _retad( A, -1, "^1" ) == "ba"

 

                example 2:

 

                A[ BEGIN_ ] = -2

 

                A[ -2 ] = a                # $5 or ^0

                A[ -1 ] = b                # $4 or ^1

                A[  0 ] = c                # $3 or ^2

                A[  1 ] = d                # $2 or ^3

                A[  2 ] = e                # $1 or ^4

                A[  3 ] = f                # $0 or ^5

 

                A[ END_ ] = 4

 

                _retad( A, -1, "^2" )        == "cba"

        in other cases `ab is always interpreted as the exact start

        index of the join-process:

 

                _reta( A, 4 )

 

                _reta( A, -3 )

 

                _reta( A, 3.14159 )

 

                _reta( A, 0 )

 

                _reta( A, "+1" )

 

 

 

`q         - quantity: numer of elements to return (index

        range)

 

 

        finally parameters determination system calculates the number

        of elements to join while joining process.

 

        `q may be present as an exact positive value (number or strnum)

                in this case number of returned elements will be equals

                to integer of `q:

 

                        number = int( `q )

 

        `q may be present as an negative value

 

                in this case its will be interpreted as: ... "return

                elements starting from determinated start index and until

                to last/first (depending on index step polarity) element

                of a given array `A.

 

                example:

 

                A[ 0 ]= a

                A[ 1 ]= b

                A[ 2 ]= c

                A[ 3 ]= d

 

                _reta( A, 1, -1 ) == "bc"

 

                _reta( A, 2, -1, -1 ) == "cb"

 

        `q may be omitted ( == "" ) in this case it's will be

        autodetected and will covers array's indexes starting from

        determinated start index and until the last/first (depending on

        index step polarity) index of a given array:

 

                if index step polarity is positive ( >= 0 ) then `q will

                be calculated for covering all elements starting from

                determinated start index until the last element of an

                array:

 

                example:

 

                A[ 0 ]= a

                A[ 1 ]= b

                A[ 2 ]= c

                A[ 3 ]= d

 

                _reta( A, 1, "", 1 ) == "bcd"

 

                if index step polarity is negative ( < 0 )  then `q will

                be calculated for covering all elements starting from

                determinated start index until the first element of an

                array:

                example:

 

                A[ 0 ]= a

                A[ 1 ]= b

                A[ 2 ]= c

                A[ 3 ]= d

 

                _reta( A, 2, "", -1 ) == "cba"

ARRAY TYPES

 

 

        COMMON array

 

                an common array is the typical post-split/patsplit array:

 

                - common array indexes is starting from index 0 or 1

 

                - all numeric indexes is present sequentially

 

                - there is no other indexes in an array except sequential

                numeric

 

                        this is except index STEP_ that's may be present in

                        a common array

 

                - the array index step is equals to: +1

 

 

        BEGIN/END(BE) array

 

                BE array is determinated by existance in an array index

                BEGIN_ or/and END_

 

                        - A[ BEGIN_ ] contains the first index in an array

                        - A[ END_ ] contains the beyond the last index of

                        an array

 

                        example:

 

                                A[ BEGIN_ ] = -2

       

                                A[ -2 ] = "a"

                                A[ -1 ] = "b"

                                A[ 0 ]  = "c"

                                A[ 1 ]  = "d"

                                A[ 2 ]  = "e"

                                A[ 3 ]  = "f"

 

                                A[ END_ ] = 4

 

                                _reta( A ) == "abcdef"

 

                        if A[ BEGIN_ ] == A[ END_ ] then BE-array is empty

 

                        - A[ STEP_ ] contains the array index step that's

                        may be positive or negative:

 

                        example:

 

                                A[ END_ ] = -3

       

                                A[ -2 ] = "f"

                                A[ -1 ] = "e"

                                A[ 0 ]  = "d"

                                A[ 1 ]  = "c"

                                A[ 2 ]  = "b"

                                A[ 3 ]  = "a"

 

                                A[ BEGIN_ ] = 3

                                A[ STEP_ ] = -1

 

                                _reta( A ) == "abcdef"

 

                        note that if STEP_ is not exist in an array then

                        array index step will be equals to: +1. this is

                        except note2

 

                        note2: if STEP_ is not exist in an array AND both

                        BEGIN_ and END_ are exist in an array AND BEGIN_ is

                        more than END_ then array index step will be equals

                        to: -1

                        example:

 

                        A[ END_ ] = -3

       

                        A[ -2 ] = "f"

                        A[ -1 ] = "e"

                        A[ 0 ]  = "d"

                        A[ 1 ]  = "c"

                        A[ 2 ]  = "b"

                        A[ 3 ]  = "a"

 

                        A[ BEGIN_ ] = 3

 

                        _reta( A ) == "abcdef"

 

                        - BE array may contains any number of the any

                        indexes

SELF3MOD

 

 

        the library have self-modification feature allowing for user to

        rebuild library source file using other constant values.

 

        the constant values are placed in the gawk commentaries.

        depending from it's values the library file source will be

        self-modificated.

 

        this feature is disabled by default but user may activate it by

        uncommenting _self3mod() function call at the BEGIN area of the

        library source file.

 

        user also must provide filepath to the library source file as

        the parameter for _self3mod()

 

        user may easily implementing the simple repeatatives in the

        source file using _self3mod feature

 

 

        _self3mod commands are placed in the gawk commentaries with

        preceeding double space characters:

 

                        ...  # x63

                        ...  # *63

 

        if such combination is found in an source line then it's

        processed as follows:

 

                whole source line is splited by the separator that is the

                double space characters

       

                each part is the self3mod command's parameter 1, 2 and so

                on

 

                the last part is the self3mod command itself

 

                note that parameters 3+ is always skipped

 

                then if last part is equals to:

 

                        ' xNumber'                - then this is the single line

                                                repeater command

 

                        or

 

                        ' *Number'                - then this is the multi line

                                                repeater command

 

                otherwise command will be skipped and source line pass to

                the result without changes

 

        single line repeater

 

                pass parameter1 "  " parameter2 "  " ( duplicates

                parameter2 "  " ) command

 

                note that parameter 1 is passed with replacement decimal

                numbers by detailed description of self3mod commands is

                currently unfinished.

HIDs

 

 

        hids is the special definition. each hid is the global variable

        that's have assigned by        the special unique value. this unique

        value may be found as the index in the global array _HID.

 

        traditionally hids have upper cased name with the single

        uderscore character at the end:

 

        the hid's usage motivation is simple:

 

                example:

 

                        let's for example you want defines in an array - the

                        subarray that will contain some list-type information

                        (just for example). but if you're naming index of the

                        subarray someway like "LIST" - it's will be good chance

                        that such index will be conflicted with the other

                        possible indexes of the original array. hid - is the

                        solution for this

 

                        we're should define in such case hid named like LIST_.

                        this mean that the global var LIST_ will be defined and

                        special unique value should to be assigned to it

 

                        now we can define subarray using content of globvar

                        LIST_ as the index of an subaray:

 

                                A[ LIST_ ]

 

        please note that hids is the part of special environment that

        this library is not includes. because of public version this

        library should operate without requiring         any external

        resources.
